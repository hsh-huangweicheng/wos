# WOS作者匹配工具

## Problem
校长需要考核各学院论文产出，图书馆工作人员需要将WOS导出的英文作者名匹配识别为本校员工的中文名，以便进行科研统计。目前手工匹配工作量大且容易出错。

## Evidence
- Assumption — needs validation via 手动统计实验

## Users
- **Primary**: 图书馆工作人员，需要统计本校员工发表的SCI论文
- **Not for**: 普通教师或学生

## Hypothesis
我们相信**拼音匹配工具**将**自动识别本校员工论文作者**供**图书馆工作人员**使用。我们将知道是对的，当**匹配结果可用于学院论文产出统计**。

## Success Metrics
| Metric | Target | How measured |
|---|---|---|
| 匹配正确率 | >90% | 抽样核对匹配结果 |
| 处理速度 | <10秒/文件 | 实际运行测试 |

## Scope
**MVP** — 分析WOS论文记录，匹配员工拼音，输出CSV

技术选型：Node.js + TypeScript（CLI工具）

代码质量要求：
- 使用 TypeScript 增加类型安全
- 代码中添加合适的注释
- 添加日志方便理解和定位问题
- ESLint + Prettier 代码规范
- 单元测试覆盖率 >80%
- 每个函数 <50 行
- 每个文件 <800 行

功能需求：
1. 支持多个WOS文件输入，文件放在 data/wos 目录下，处理该目录下所有文件
2. 解析WOS savedrecs.txt格式（AU、AF、UT、C1等字段）
   - AF列：每个作者占一行，按顺序排列确定"第几作者"
   - C1列：每行 `[作者1; 作者2; 作者3] 机构名称`，作者用分号分隔
3. 机构过滤：从C1列匹配本校作者，支持正则表达式，忽略大小写
   - C1字段格式：每行以 `[` 开头，方括号内是作者列表（用分号分隔），方括号后面是机构名称
   - 例如：
     ```
     C1 [作者1; 作者2; 作者3] 机构名称1
        [作者4; 作者5] 机构名称2
     ```
   - 匹配逻辑：遍历C1每行，用正则匹配机构名，匹配成功则该行方括号内的作者都是本校作者
4. 加载机构配置CSV（英文名称，正则表达式）
5. 加载员工配置CSV（姓名、部门、拼音，拼音用分号分隔）
6. 预处理：
   - 英文和拼音都转小写
   - 非字母数字空格的其他字符 → 替换成空格
   - 连续多个空格 → 只保留一个
   - 头尾空格去掉
7. 拼音匹配算法：只支持全匹配，使用AF列
8. 相同拼音处理：一个拼音匹配出多个中文名时，用方括号括起来
9. 第几作者：通过AF列确定作者顺序
10. 未匹配作者：需要输出，中文名为空
11. 输出CSV（UT、作者英文、作者中文、第几作者），文件名为"结果.csv"，覆盖方式
12. 输出结果中 UT号+英文姓名 组合唯一（去重）

编码：所有文件采用 UTF-8

**Out of scope**
- 图形界面/Web页面
- 数据库存储
- 复杂的模糊匹配算法（如部分匹配 "Zhang, X."）
