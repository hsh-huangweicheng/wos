# WOS 解析应用 - 实现计划

## Context

用户需求：开发纯前端 WOS 解析应用，部署到 Vercel，用于分析指定机构的作者。

### 需求澄清结果

| 类别 | 内容 |
|------|------|
| 框架 | React 18 + TypeScript |

| 数据量 | 上千条记录，需要分页和性能优化 |

## WOS 解析器设计（独立能力）

WOS 文件格式：每个字段以标签开头（如 FN、VR、AU），字段值可跨多行，直到遇到下一个标签。

### WosRecord（WOS 记录 - 完整字段）

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| ut | string | 唯一标识 | WOS:001710458900011 |
| au | string[] | 作者缩写 | ["Tang, C", "Li, QP"] |
| af | string[] | 作者全名 | ["Tang, Chao", "Li, Qingpu"] |
| ti | string | 标题 | The autonomy paradox... |
| so | string | 期刊名 | PLOS ONE |
| la | string | 语言 | English |
| dt | string | 文献类型 | Article |
| c1 | string | 作者地址 | [Tang, Chao] Jinling Inst... |
| py | number | 出版年 | 2026 |
| di | string | DOI | 10.1371/journal.pone.0336166 |

### C1 字段解析结果

```typescript
interface C1Line {
  authors: string[];    // ["Tang, Chao", "Li, Qingpu"]
  institution: string;  // "Jinling Inst Technol, Sch Animat, Nanjing..."
}
```

## 数据对象设计

### Institution（目标机构）

| 字段 | 类型 | 说明 |
|------|------|----|
| name | string | 唯一，中文名（如：金陵科技学院） |
| pattern | string | 用于匹配WOS，C1字段中的机构 |

### Employee（员工）

| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 中文名（如：刘志远） |
| department | string | 部门（如：动画学院） |

### MatchRule（匹配规则）

| 字段 | 类型 | 说明 |
|------|------|------|
| targetName | string | 目标中文名（机构或员工） |
| pattern | string | 匹配规则（正则表达式，不区分大小写） |
| type | "institution" | "employee" | 规则类型，枚举值 |

### UtDepartment（UT与部门的关系）

| 字段 | 类型 | 说明 |
|------|------|------|
| ut | string | WOS 记录 UT 号 |
| department | string | 用户选择的部门 |
| updatedAt | string | 更新时间 |

## 页面4：部门分析（详细设计）

**路由**：`/analysis/institution/department`

### 列表展示字段

| 字段 | 说明 |
|------|------|
| UT | WOS 记录的唯一标识 |
| 标题 | 文章标题 |
| 机构 | WOS 中的机构英文名 |
| 作者 | WOS 中的作者英文名 |
| 机构中文 | 根据配置匹配到的机构中文名（可能为空/1个/多个） |
| 作者中文 | 根据配置匹配到的员工中文名（可能为空/1个/多个） |
| 部门 | 根据配置匹配到的部门（可能为空/1个/多个） |

### 筛选选项

- **Checkbox**：仅显示已匹配到中文的机构数据
  - 勾选：只显示 机构中文 不为空的记录
  - 不勾选：显示所有记录

### 导出功能

- 页面内提供 CSV 导出按钮

### 匹配逻辑说明

**输入**：
- C1 每行是一个机构，包含 1+ 个作者
- WOS 中的名称全是英文或拼音
- 用户只有作者中文和相应的部门

**问题场景**：
1. WOS 作者英文可能匹配到多个作者中文（拼音变体相同）
2. 不同作者中文可能对应相同的作者英文/拼音
3. 相同作者中文可能有多个部门

**匹配结果展示规则**：

| 匹配情况 | 显示 |
|----------|------|
| 匹配到 0 个 | 显示为空 |
| 匹配到 1 个 | 正常显示该值 |
| 匹配到多个 | 显示多个名称（用逗号分隔），供用户选择 |

### 用户交互规则

**未匹配（为空）时**：
- 下拉框显示所有员工姓名
- 支持搜索过滤
- 用户选择一个后，更新显示并保存/更新匹配规则

**匹配到多个时**：
- 只展示这几个匹配到的名称
- 用户点击可重新选择
- 选择后更新显示并保存/更新匹配规则

**用户选择后**：
- 可以修改选择
- 选择后记录 MatchRule（作者英文 → 作者中文）
- 选择后记录 UtDepartment（UT号 → 部门）

### 重名员工处理

- 相同作者中文可能有多个部门
- 用户选择部门后，记录 UT 与部门的关系
- 当该 UT 的作者发生变化时，清空这个关系
